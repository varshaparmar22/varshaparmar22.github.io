AI-Powered WhatsApp Healthcare Assistant – Serverless Agent-Based System

Technologies: AWS Lambda, DynamoDB, LangGraph, LLM (RAG), WhatsApp Business API

About the Project

AI-Powered WhatsApp Healthcare Assistant is a production-grade, multilingual (English/Gujarati) conversational system designed to automate hospital-patient interactions through WhatsApp. The system leverages AWS serverless architecture and LLM-based agent orchestration to enable appointment booking, symptom-based doctor recommendation, medical Q&A, and secure report summarization. Built with scalability, privacy, and healthcare-safe responses in mind, the architecture follows an event-driven, stateless execution model with persistent conversation state management.

🧠 Key Objectives

  • Design a scalable WhatsApp-based healthcare assistant with multilingual support.
  • Implement agent-based routing for appointment booking, report analysis, and medical Q&A.
  • Ensure secure document handling with sandboxed processing.
  • Maintain conversation continuity using stateless Lambda execution with external state storage.
  • Optimize for cost-efficiency, reliability, and healthcare compliance.

⚙️ Tech Stack

  • Cloud Architecture: AWS Serverless (Lambda-based execution model)
  • State Management: Amazon DynamoDB with TTL-based auto-expiry
  • Agent Orchestration: LangGraph-based routing and decision logic
  • LLM Integration: RAG-based contextual responses
  • Document Processing: Isolated Lambda-based sandbox execution
  • Messaging Platform: WhatsApp Business API (1-to-1 communication)
  • Event Handling: Event-driven asynchronous workflows
  • Language Support: English & Gujarati (template-driven deterministic responses)

🔍 System Architecture

  • 1. Message Ingestion: Each incoming WhatsApp message triggers a stateless AWS Lambda invocation.
  • 2. Conversation State Management: Conversation context (intent, step, language preference) is stored in DynamoDB. Each Lambda reads and updates state to maintain multi-step workflows.
  • 3. Agent Routing Layer: LangGraph determines user intent (appointment, report, medical Q&A, escalation). Deterministic keyword routing with LLM fallback ensures safe intent detection.
  • 4. Appointment Booking Flow: Multi-step interaction collects symptoms, recommends doctors, and confirms booking. State transitions handled explicitly via step tracking.
  • 5. Secure Document Processing: Uploaded medical reports are stored in isolated storage and processed in sandboxed Lambda environments. Raw files are never exposed to the LLM; only sanitized extracted text is used.
  • 6. Asynchronous Workflow Handling: Long-running tasks (report processing) use event-driven updates and user notifications.
  • 7. State Lifecycle Management: DynamoDB TTL automatically cleans inactive conversation records.

🔬 Key Technical Achievements

  • 📩 Event-Driven Architecture — Each WhatsApp message handled as independent stateless execution.
  • 🗂 Conversation State Persistence — Enabled scalable multi-step workflows without session management.
  • 🔐 Secure File Isolation — Implemented sandboxed Lambda execution for safe document summarization.
  • 🌐 Multilingual Support — Built deterministic template system for English and Gujarati responses.
  • ⚡ Cost Optimization — Reduced orchestration overhead by using lightweight code-driven workflow instead of heavy state machines.
  • 📈 Production Scalability — Designed horizontally scalable architecture supporting high concurrent users.

🧮 Operational & Reliability Measures

  • State expiry using DynamoDB TTL to prevent stale sessions.
  • Graceful failure handling with explicit FAILED state updates.
  • Idempotent processing to prevent duplicate bookings or repeated executions.
  • Separation of responsibilities: Chat handling Lambda vs Document processing Lambda.
