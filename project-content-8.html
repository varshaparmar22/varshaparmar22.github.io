MeetingMind AI – Production-Grade Meeting Intelligence SaaS

Technologies: FastAPI, PostgreSQL, SQLAlchemy, AWS S3, AWS Transcribe, Whisper, Ollama, Claude, GPT-4, AWS Bedrock, Stripe, Next.js, React, Docker, GitHub Actions

About the Project

MeetingMind AI is a production-grade, multi-tenant SaaS platform that turns business meetings into actionable intelligence. Teams upload audio or video, and the platform automatically transcribes the recording, summarizes the discussion, and extracts action items, decisions, and key topics — all organized inside collaborative workspaces with search and team management.

It is built around an async FastAPI backend with a PostgreSQL-backed job queue, a provider-agnostic LLM layer, a switchable transcription engine, and full Stripe billing with usage metering. The frontend is a modern Next.js / React app.

🧠 What It Does

  • Transcribes meetings with OpenAI Whisper (local dev) or AWS Transcribe with speaker labels (production) — switchable by a single environment flag.
  • Summarizes discussions through a configurable LLM (Ollama, Claude, GPT-4, or Bedrock).
  • Extracts action items, owners, deadlines, decisions, and key topics into structured data.
  • Organizes meetings in multi-tenant workspaces with role-based team collaboration.
  • Meters & bills usage with Stripe subscriptions and plan-limit enforcement.

⚙️ Tech Stack

  • Backend: FastAPI, Uvicorn, SQLAlchemy 2.0, Alembic, JWT (python-jose), bcrypt
  • Data: PostgreSQL 16 — users, workspaces, members, meetings, subscriptions, async job queue, cost & transcript cache tables
  • Storage: AWS S3 (audio/video) with a disk-backed local-storage fallback for dev
  • Transcription: Whisper (local / OpenAI API) and AWS Transcribe, behind a factory chosen via TRANSCRIPTION_SERVICE
  • LLM Layer: a provider abstraction (BaseLLMProvider + LLMFactory) over Ollama (llama3.1, free local default), Claude, OpenAI GPT-4/3.5, and AWS Bedrock
  • Billing: Stripe — Checkout, Customer Portal, signature-verified webhooks, currency-aware pricing (USD/INR)
  • Frontend: Next.js + React, Tailwind, React Query, Zustand, react-hook-form + Zod, TypeScript
  • DevOps: Docker (lean python:3.12-slim image), Docker Compose, GitHub Actions CI/CD, AWS ECR, EC2 + RDS, slowapi rate limiting, Pytest (90 tests)

🔍 System Architecture

  • User layer (Next.js): auth, meeting upload, dashboard, and workspace management talk to the API over HTTPS with JWT access/refresh tokens.
  • API layer (FastAPI): versioned /api/v1 routes for auth, workspaces, members, meetings, and billing — workspace-scoped via an X-Workspace-ID header.
  • Service & data-access layer: Auth, Workspace, Meeting, S3, Job Queue, and Worker services backed by clean repositories.
  • Data & external layer: PostgreSQL, AWS S3, the transcription engines, and the LLM providers.

🔄 Meeting Processing Pipeline

  • 1. Upload — validate format (mp3, wav, m4a, mp4), create the meeting record, store in S3.
  • 2. Queue — insert a job into the PostgreSQL async_jobs table (no Redis needed).
  • 3. Process — a background worker polls every 5 seconds and marks the job running.
  • 4. Transcribe — the factory selects Whisper or AWS Transcribe; a SHA256 transcript cache avoids re-paying for identical audio (~20–30% savings).
  • 5. Summarize — the LLM returns a summary, action items, and key topics, parsed tolerantly to survive model quirks.
  • 6. Complete — results are saved and the meeting status flips to completed (typically 1–15 minutes end-to-end).

👥 Multi-Tenancy & Billing

  • Workspaces & roles: owner / admin / member RBAC — members view, admins manage members and settings, and the last owner can never be demoted or removed.
  • Subscription tiers: Free (5 meetings/mo, 500 MB), Pro, Business, and Enterprise — each with monthly meeting and storage limits.
  • Enforcement: uploads check plan limits and return HTTP 402 with an upgrade prompt when exceeded.
  • Stripe sync: webhooks keep the subscriptions table in step with Checkout and Portal events.

🔬 Key Technical Achievements

  • 🧩 Pluggable engines — one interface over four LLM providers and two transcription backends, switchable per environment with zero code changes.
  • 🧵 Redis-free job queue — durable async processing built directly on PostgreSQL with a polling worker and proper failure states.
  • 💳 Tested Stripe billing — end-to-end Checkout → subscription → webhook sync verified against a live test account, including India RBI e-mandate handling and INR pricing.
  • 💰 Cost optimization — transcript caching, tiered model usage, and Claude-over-GPT-4 summaries cut summary costs up to ~87% (~$1,300+ projected annual savings).
  • 🧪 Quality gates — a 90-test Pytest suite, per-endpoint rate limiting, and a 4-stage GitHub Actions pipeline (lint → test → ECR push → EC2 deploy).
